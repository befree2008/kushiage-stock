> 📋 这是老王和星月每周配合的标准操作流程

---

## 🔄 完整流程循环

```
┌───────────────────────────────────────────────────────────┐
│  1️⃣  每周固定日（比如周一）                                 │
│      星月提醒："老王，本周销售数据该更新了！"                │
└───────────────────────────────────────────────────────────┘
                             ↓
┌───────────────────────────────────────────────────────────┐
│  2️⃣  老王发美团管家导出的 Excel                              │
└───────────────────────────────────────────────────────────┘
                             ↓
┌───────────────────────────────────────────────────────────┐
│  3️⃣  星月自动处理：                                          │
│      ✅ 扣减库存（带日期防呆）                               │
│      ✅ 生成补货建议清单                                     │
└───────────────────────────────────────────────────────────┘
                             ↓
┌───────────────────────────────────────────────────────────┐
│  4️⃣  老王：看建议 → 下单 → 收货 → 拍进货单发我              │
└───────────────────────────────────────────────────────────┘
                             ↓
┌───────────────────────────────────────────────────────────┐

│      🔍 识别进货单 → 发表格给老王确认                        │
│      ✅ 确认后 → 自动入库更新库存                            │
└───────────────────────────────────────────────────────────┘
                             ↓
                         回到步骤 1
```

---

## 📅 详细步骤说明

---

### **第一步：导出销售数据（老王）**

**触发时机：** 每周固定日（比如周一早上），星月会主动提醒。

**老王操作：**
1. 登录美团管家后台
2. 导出上周（或指定日期范围）的销售统计 Excel
3. 发给星月

---

### **第二步：扣库存 + 补货建议（星月全自动）**

**系统自动完成：**

| 步骤 | 动作 | 防呆保护 |
|------|------|---------|
| 1 | 从 Excel 自动提取日期范围 | 日期不连续会提醒确认 |
| 2 | 美团菜品名 → SKU 智能匹配 | 未匹配会标注出来 |
| 3 | 按 SKU 汇总销量 | |
| 4 | 自动扣减对应库存 | 重复日期会强制阻止 |
| 5 | 按安全库存计算补货建议 | |

**老王会收到：**
```
📊 本周销售扣减完成！
├── 统计周期：2026-05-11 ~ 2026-05-17 (7天)
├── 扣减 SKU：61 个
└── 🛒 补货建议：
    SKU009 鸡肉小串     缺 3000 串 → 建议进 3 箱
    SKU005 玉米粒小串   缺 1500 串 → 建议进 2 箱
    SKU007 纸豆皮小串    缺 1000 串 → 建议进 1 箱
    ...
```

---

### **第三步：收货拍照（老王）**

**老王操作：**
1. 看星月给的补货建议，调整数量后下单
2. 货送到后，把进货单拍照发给星月

---

### **第四步：进货单识别 + 入库（星月全自动）**

**实际工作流（v3.0，2026-05-17 起启用）：**

| 步骤 | 谁 | 动作 |
|------|-----|------|
| 1 | 老王 | 把进货单图片发给星月 |
| 2 | 星月 | 用 image 工具 OCR 识别（按 TOOLS.md 规则，**必须**走 imageModel，不可用对话模型直接看图） |
| 3 | 星月 | 查 `data/sku_dictionary.csv` 把每件货品折算成 **每箱基准数量**（base_unit） |
| 4 | 星月 | 生成 `data/order_<单号>.csv`（CSV 模板见下） |
| 5 | 星月 | 跑 `--dry-run`，把匹配表 + 入库前后数量发给老王确认 |
| 6 | 老王 | 核对后回复 `y` 或具体修正 |
| 7 | 星月 | 跑真实入库（`--yes` 跳过交互），自动备份 + 写日志 |

**进货 CSV 格式（`data/order_<单号>.csv`）：**
```csv
item_name,boxes,per_box_base,note
卤油,1,10000,20袋/箱 × 500g/袋
台湾无骨鸡柳,1,10,10袋/箱 × 1kg/袋
鸡叉骨,1,15,6袋/箱 × 2.5kg/袋
```
- `item_name`：进货单货品名称（与 `sku_dictionary.sku_name` 或 `aliases` 匹配）
- `boxes`：箱数
- `per_box_base`：每箱折算成 `base_unit` 的数量（重点！）
- `note`：进货单上的规格说明（写日志用）

**命令行：**
```bash
cd work/kushiage-stock-github

# 演练：只匹配不入库
python3 purchase_order_import.py data/order_OCK2026051309342.csv \
  --order-no OCK2026051309342 --dry-run

# 真实入库（确认无误后）
python3 purchase_order_import.py data/order_OCK2026051309342.csv \
  --order-no OCK2026051309342 --yes
```

**自动产生的副作用：**
- 📁 备份 `data/inventory.json` → `data/inventory.json.before_purchase_<时间戳>`
- 📜 日志写入 `data/inventory_log.json`（每件一条 ADD 记录，含原因）
- 📊 终端打印入库前后库存清单

**老王会收到确认表格示例（dry-run 输出）：**
```
进货单货品           匹配 SKU   SKU 名称        箱数    入库数    单位   当前库存
卤油               SKU039   卤油            1   10000.00 g     10000.00
小串纸桶（小号）       SKU043   小串纸桶（小号）    1     500.00 个       250.00
紫苏孜然撒粉         SKU036   紫苏孜然撒粉      1   10000.00 g      1000.00
卤山川巴掌大鸡排       SKU018   卤山川巴掌大鸡排    1     120.00 串       100.00
台湾无骨鸡柳         SKU033   台湾无骨鸡柳      1      10.00 kg        1.00
...（共 11 行）
✅ 演练完成，全部命中。去掉 --dry-run 执行真实入库。
```

**SKU 匹配规则（按优先级）：**
1. 精确匹配 `sku_name`
2. 精确匹配 `aliases`（字典里可选列，分号 / `;` / `|` 分隔）
3. 子串包含匹配（双向）

匹配多个或匹配不到 → 脚本**不会写库存**，直接退出报错，要先在 `sku_dictionary.csv` 加 `aliases` 或修改进货单 CSV。

## 🛡️ 安全防呆机制

| 场景 | 系统动作 |
|------|---------|
| **重复扣减**（同一日期范围扣了又扣） | ❌ 强制阻止，给出提示 |
| **日期不连续**（上次扣到5.10，这次从5.12开始） | ⚠️ 警告提醒，确认后继续 |
| **进货单有新品**（系统里没有的SKU） | ⚠️ 标注"需确认"，不自动入库 |
| **所有操作** | 📝 完整记录日志，随时可查 |

## 💡 老王小贴士

1. **进货单照片越清晰，识别越准**，拍的时候尽量平放、光线好
2. **日期范围尽量连续**，避免漏掉某几天的销量没扣
3. **有新品进货**提前告诉我，我把SKU加到字典里下次就能自动识别了
4. **想单独查库存/调账**随时说，我给你操作
